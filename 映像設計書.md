## Gymty（体育館検索アプリ）── 直接型 ×7

### 仮説の前提
memory.mdより：Gymtyは「不満訴求」より「ユーザーが今まさに検索しそうな言葉」で始める方が強い。「地域名＋日時＋種目」の具体性が刺さる。広い便利訴求より「今すぐ使う具体状況」が有効。

---

### パターン①（メモ原案）サークル幹部の睡眠不足

| 構成 | 内容 |
|------|------|
| 起 | 大学のサークル幹部が授業中に机に突っ伏して寝ている。友達に「また寝てんじゃん」と突っ込まれる |
| 承 | 眠い理由は、練習場所の体育館を確保するために夜中まで何個ものWebサイトを巡回して空き状況を調べていたから。画面には大量のタブが開いたブラウザ |
| 転 | Gymtyを知り、スマホ1つで一括検索。数タップで空き体育館が見つかる画面を映す |
| 結 | テロップ：「体育館探しなら Gymty」。幹部がぐっすり寝てる映像で締め |

**ターゲット**：大学の体育系サークル幹部（20歳前後）
**訴求仮説**：「体育館予約のために睡眠を削っている」というあるあるの不満に共感させ、時短ツールとしてのGymtyを提示する

## GoGoBombs（ごごばく）── 間接型 ×7

### 仮説の前提
memory.mdより：「PC不要」の広い訴求より「配置・二択・自己認識」が強い。「エイムは関係ない、頭脳で勝つ」が刺さる。ゲームっぽい演出そのものより「視聴者が自分を投影できる問い」が有効。

---

### パターン①（メモ原案）PCがなくて話についていけない

| 構成 | 内容 |
|------|------|
| 起 | 昼休み、友達がバロラントの話で盛り上がってる。主人公はPC持ってないからついていけない。愛想笑いで相槌を打っている |
| 承 | みんなが「あそこでスモーク炊いて〜」「フェイクでA行って〜」と話してるのに、主人公は何のことかわからない。だんだん会話から外れていく |
| 転 | ある日、通学中にスマホでGoGoBombsをやっている。配置を考える、攻め方を選ぶ。気づかないうちにバロラントの戦略がわかるようになってきた |
| 結 | 次の昼休み、友達の話に「それ、Bサイト裏取りした方がよくない？」と自然に入れるようになっている。テロップ：「PCがなくても、話についていける」 |

**ターゲット**：PCを持っていないがバロラント好きの友達がいる学生（15〜20歳）
**訴求仮説**：「友達の輪に入れない」という感情的な痛みからスタート。ごごばくを「理解度アップツール」として位置づける

## ガラシャ（ガラケー風カメラアプリ）── 直接型＋間接型 ×7

### 仮説の前提
ガラシャは「ガラケーのデザインで撮った写真がガラケー風になるアプリ」。ターゲットは大きく3層：①レトロ・Y2K好きの若い女性、②懐かしさを感じたい30〜40代、③写真加工が好きなSNSユーザー。直接型と間接型の両方で散らす。

---

### パターン①（直接型）インスタの写真、みんな同じに見える問題

| 構成 | 内容 |
|------|------|
| 起 | 女子大生がインスタのフィードをスクロールしている。友達の投稿がみんな同じようなフィルター、同じような雰囲気。「なんかどれも似てるな…」 |
| 承 | 自分も撮ってみるけど、同じフィルターをかけると同じ仕上がりになる。「おしゃれだけど、個性がない」とモヤモヤ |
| 転 | ガラシャで撮ってみる。ガラケー風の画質・フレーム・日付表示。明らかに他と違う質感の写真ができる |
| 結 | インスタに投稿したら「これどうやって撮ったの！？」とDMが来る。テロップ：「みんなと違う1枚、ガラシャ」 |

**ターゲット**：インスタで個性を出したい女子大生（18〜22歳）
**訴求仮説**：「みんな同じフィルター問題」という不満から入り、差別化ツールとしてガラシャを提示（直接型）

## 毎メモ（毎日の調子記録アプリ）── 間接型 ×7

### 仮説の前提
毎メモは「毎日の調子を記録するアプリ」。ターゲットは大きく3層：①メンタルの波に悩む人、②常に最高のパフォーマンスを出したい人、③記録・振り返りが好きな人。すべて間接型で、余韻を残す構成にする。

---

### パターン①　病んでた自分が立ち直れた理由

| 構成 | 内容 |
|------|------|
| 起 | 大学生の女の子。朝起きられない、何もやる気が出ない。布団の中でスマホだけ触ってる日が続いている |
| 承 | ある日、なんとなく毎メモで「今日の調子：2/10」と記録してみる。次の日も、その次の日も。ただ数字を入れるだけ |
| 転 | 2週間後、記録を振り返ると「水曜だけ少しマシ」というパターンに気づく。水曜はバイトがある日。「人と話す日は調子いいんだ」と自分でも知らなかった法則を見つけた |
| 結 | 意識的に人と会う日を増やした。調子の波が緩やかになっている記録画面。テロップ：「気づけなかったことに、気づける」 |

**ターゲット**：メンタルの波に悩む若者（18〜25歳）
**訴求仮説**：「毎日記録するだけで自分のパターンが見える」という価値を感情ストーリーで伝える

### パターン②　なぜか月曜に調子が悪いエンジニア

| 構成 | 内容 |
|------|------|
| 起 | 20代後半のエンジニア。毎週月曜だけコードレビューでミスを指摘される。「月曜はなぜかダメなんだよな…」 |
| 承 | 毎メモで調子を記録し始める。体調、睡眠、食事、気分を毎日つけていく |
| 転 | 1ヶ月分のデータを見ると、日曜の夜に寝るのが遅い→月曜の睡眠時間が平均2時間少ない、とわかった。日曜の夜だけ早く寝るようにした |
| 結 | 月曜のパフォーマンスが安定。「最近月曜も調子いいね」と同僚に言われる。テロップ：「調子に理由があった」 |

**ターゲット**：パフォーマンスにこだわる社会人（25〜35歳）
**訴求仮説**：「なんとなく調子が悪い日」に原因があることをデータで可視化する価値を見せる

### パターン③　受験生の体調管理

| 構成 | 内容 |
|------|------|
| 起 | 高校3年生の受験生。毎日勉強しているが、日によって集中できる日とできない日の差が激しい。模試の結果もバラバラ |
| 承 | 母親に「ちゃんと体調管理しなさい」と言われるが、何をどう管理すればいいのかわからない。毎メモで「勉強時間・集中度・睡眠・体調」を記録し始める |

**ターゲット**：受験生とその保護者（16〜18歳）
**訴求仮説**：「勉強法」ではなく「体調管理」が成績を左右するという気づき。記録の力を成果と結びつける

### パターン④　彼女の「なんかイライラする」の正体

**訴求仮説**：生理周期と気分の関連に気づくツールとして。デリケートな話題だが共感度が高い
